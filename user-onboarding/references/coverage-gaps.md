# User Onboarding Coverage Gaps

Use this file to park onboarding patterns that are promising but not yet strong enough to become reusable agent rules.

## Candidate Gaps

- **B2B admin and team-invite onboarding.** `driving-momentum.md` mentions internal champions and team adoption, but the skill lacks accepted examples for admin dashboards, invite sequencing, permission prompts, and team rollout states.
- **AI-native first-run experiences.** The body references inverted onboarding and free play, but examples should route through `ai-experience-design` before this skill promotes rules for prompt surfaces, generated previews, memory consent, or AI setup questions.
- **Lifecycle email exemplars.** The Mount Rushmore categories are present, but there are no before/after examples tied to source evidence for welcome, progress, come-back, or trial-expiring emails.
- **Empty-state thresholds for engagement metrics.** `empty-states.md` includes the zero-count warning, but this needs more product examples before becoming a general rule for all metrics.
