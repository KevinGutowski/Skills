# Coverage Gaps

Use this file for org-influence scripts, career guidance, and business-case patterns that need stronger evidence before becoming reusable guidance.

## Candidate gaps

- **Promotion packet exemplars:** The skill has strong SBI/wins-doc guidance, but reusable promotion examples should come from accepted review artifacts with identifying details removed.
- **Stakeholder scripts by role:** PM, engineering, executive, design manager, and finance objections need sourced or tested scripts before they become templates.
- **AI-era org operating model:** Claims about designers shipping PRs, CEO-in-the-tool diagnostics, and Claude-skill review logging decay quickly; keep examples dated and scoped.
- **Metric/value claims:** Business-case examples should state whether numbers are project data, directional estimates, or borrowed examples.
- **Boundary with `product-decision-making`:** If the problem is poor strategy, route there; this skill should not turn weak product judgment into a better pitch.
