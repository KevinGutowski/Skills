# Coverage Gaps

Use this file for process candidates that should not be flattened into universal rules.

## Candidate gaps

- **School-selection examples:** Need accepted examples showing when to choose Basecamp Shape Up, Linear, Martian design sprint, or metrics-led product strategy.
- **Betting-table artifacts:** Add anonymized pitch, appetite, no-go, and scope-hammering examples only after real cycles or reviewed examples are available.
- **AI-agent workflow:** The skill does not yet say how coding/design agents change shaping, betting, or building; add rules only from observed repeatable workflows.
- **Metrics vs taste conflict:** Add examples where a team deliberately chose data, taste, or fixed appetite and recorded the outcome.
- **Deterministic checks:** Possible future checks include pitch completeness, appetite presence, risks, circuit breakers, and owner/decider clarity, but only if artifact formats stabilize.
