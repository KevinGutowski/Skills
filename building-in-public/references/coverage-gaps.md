# Coverage Gaps

Use this file for building-in-public guidance that depends on current platforms, audience data, or accepted launch examples.

## Candidate gaps

- **Platform mechanics:** Algorithms, formats, safe zones, post types, and newsletter/social tools drift quickly; verify current platform behavior before tactical advice.
- **Launch exemplars:** Add examples only when prototype, post, audience response, product change, and outcome are captured.
- **Audience metrics:** Avoid universal thresholds for comments, waitlists, conversion, or retention; use project baselines and niche size.
- **Ethics/privacy:** Sharing work-in-progress must not expose client/user data, confidential plans, or misleading prototypes.
- **Boundary with `devtools`:** Developer-tool landing pages, OSS growth, and dev PMF belong in `devtools`; this skill owns maker/audience operating rhythm.
