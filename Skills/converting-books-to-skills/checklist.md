# Checklist: Before Shipping Skills

Use this checklist to verify your book-to-skills conversion is complete.

---

## Analysis Phase

- [ ] Mapped the book's structure (parts, chapters)
- [ ] Identified major phases/workflows
- [ ] Listed core mindsets/philosophies
- [ ] Noted concrete examples and case studies
- [ ] Identified key vocabulary/trigger words

## Bundling Phase

- [ ] Grouped chapters into phase-based skills (not 1:1)
- [ ] Chose skill names (use gerund form: `shaping-work`)
- [ ] Decided on file naming convention

## SKILL.md Quality

- [ ] Under 500 lines
- [ ] Sections follow book's chapter order
- [ ] Each section has 1-2 sentence narrative
- [ ] Each section has decision framework (table or list)
- [ ] Each section has one grounding example
- [ ] Each section links to a reference file
- [ ] No orphan sections (all have corresponding reference files)

## Reference Files Quality

- [ ] One file per major concept
- [ ] Includes narrative context ("why this matters")
- [ ] Multiple concrete examples from the book
- [ ] Anti-patterns and common mistakes included
- [ ] Subjective experience described where relevant
- [ ] No nested references (one level deep only)
- [ ] Consistent file naming (lowercase, hyphens)

## Descriptions

- [ ] Says what the skill does
- [ ] Says when to use it
- [ ] Includes key trigger words
- [ ] Written in third person
- [ ] Under 1024 characters

## Mindset Capture

- [ ] Core philosophies identified
- [ ] Dedicated reference files for each mindset
- [ ] Mindsets referenced in SKILL.md (not just buried in references)
- [ ] Contrast with conventional approaches explained

## Voice and Style

- [ ] Preserves book's teaching style (principle → example → principle)
- [ ] Includes memorable quotes where appropriate
- [ ] Active voice throughout
- [ ] Consistent terminology
- [ ] Concrete over abstract

## Testing

- [ ] Skill triggers on relevant keywords
- [ ] Internal links navigate correctly (not to external pages)
- [ ] Content answers common questions about the topic
- [ ] SKILL.md is scannable (can find answers quickly)
- [ ] Reference files provide sufficient depth
- [ ] No broken links

---

## Quick Metrics

| Metric | Target |
|--------|--------|
| SKILL.md lines | < 500 |
| Reference file lines | 50-250 each |
| Total reference files | 5-15 per skill |
| Description length | 100-300 characters |
| Examples in SKILL.md | 1 per section |
| Examples in reference files | 2-5 per file |

---

## Final Review Questions

1. **If someone asks about [topic], will this skill help Claude give a good answer?**
2. **Can I find information quickly by scanning SKILL.md?**
3. **Do the reference files add value beyond what's in SKILL.md?**
4. **Does the skill capture the book's ideology, not just its procedures?**
5. **Would the book's author recognize their work in this skill?**
