# Coverage Gaps

Use this file for principle-level candidates that are not ready to become reusable design law.

## Candidate gaps

- **Current HIG wording:** The local skill is a reasoning companion. Re-check Apple's live `design-principles` HIG page before quoting exact principle names or wording.
- **Project-specific facets:** The skill recommends product-specific facets, but durable examples need accepted critiques from real projects before becoming exemplars.
- **AI-era critique examples:** Add examples for "plausible artifact is not solved problem" only after repeated reviews show the same failure mode.
- **Deterministic checks:** Principle critique is mostly judgment. Mechanical checks should stay in tactical skills unless a principle maps cleanly to a stable rule.
- **Conflict decisions:** Add good/bad examples only when a real trade-off between principles was accepted and the deciding context is recorded.
